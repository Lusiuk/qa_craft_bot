# qa_craft_bot

**qa_craft_bot** — Telegram‑бот для создания и прохождения тестов.  
Роли:
- **Автор** — создаёт тесты, добавляет/редактирует вопросы, получает код доступа и результаты прохождения.
- **Ученик** — вводит код доступа, проходит тест, видит результат; автор получает уведомление о результате.

Проект учебный, но с реальным end‑to‑end сценарием: автор публикует тест → ученик проходит → автор получает отчёт.

---

## Возможности

### Автор
- Создание теста: **название + описание**.
- Добавление вопросов с вариантами ответов (один правильный).
- Просмотр своих тестов (**название → описание → код доступа**).
- Редактирование теста:
  - изменить название / описание
  - редактировать вопрос (текст + варианты + правильный)
- Удаление теста (soft delete).

### Ученик
- Указать имя пользователя.
- Ввести код доступа и начать тест.
- Отвечать на вопросы.
- Получить итог: количество правильных, процент, время прохождения.
- Пройти тест заново или выйти.

## Требования

- Ruby **3.3+** (у нас тестировалось на Ruby 4.0.2)
- Bundler
- Ngrok (для webhook‑режима)
- Telegram Bot Token (BotFather)

## Установка

1) Склонируйте репозиторий и перейдите в папку проекта:

```bash
git clone https://github.com/Lusiuk/qa_craft_bot.git
cd qa_craft_bot
```

2) Установите зависимости:

```bash
bundle install
```

3) Подготовьте базу (SQLite):

```bash
bundle exec ruby db/setup.rb
```

## Настройка `.env`

Создайте файл `.env` в корне проекта:

```env
TELEGRAM_BOT_TOKEN=XXXXXXX
NGROK_AUTHTOKEN=XXXXXXX
NGROK_API=http://localhost:4040
```

- `TELEGRAM_BOT_TOKEN` — обязателен.
- `NGROK_AUTHTOKEN` — обязателен.
- `NGROK_API` — необязателен (по умолчанию `http://localhost:4040`).

## Как запустить

Проект рассчитан на запуск в webhook‑режиме через ngrok.

### Windows (PowerShell)

Рекомендуемый способ — через `start_windows.ps1` (лежит у вас в `bin/`):

```powershell
powershell -ExecutionPolicy Bypass -File .\bin\start_windows.ps1
```

Скрипт:
- остановит старые процессы (ngrok/ruby),
- поднимет `ngrok http 4567`,
- запустит Sinatra (`sinatra_app.rb`),
- обновит webhook через `app/update_webhook.rb`.

### macOS / Linux (bash)

Рекомендуемый способ — через `start.sh` (лежит у вас в `bin/`):

Фанкционал аналогичен функционалу на Windows

```bash
bash start.sh
```

## Как пользоваться (сценарий)

### 1) Автор создаёт тест
1. Нажмите **«Я автор»**
2. **«Создать тест»**
3. Введите **название**
4. Введите **описание** (или `-`, чтобы пропустить)
5. Добавьте вопрос:
   - текст вопроса
   - варианты (каждый с новой строки)
   - номер правильного варианта
6. Добавьте следующий вопрос или нажмите **«Завершить»**

После создания бот выдаст **код доступа** — его передают ученикам.

---

### 2) Автор смотрит свои тесты
1. **«Я автор»**
2. **«Мои тесты»**

Формат списка:

- `ID — Название`
- `Описание` (если задано)
- `Код доступа`

---

### 3) Ученик проходит тест
1. Нажмите **«Я ученик»**
2. (Опционально) **«Задать имя пользователя»**
3. **«Ввести код доступа»**
4. Введите код
5. Нажмите **«Начать»**
6. Отвечайте на вопросы кнопками

После завершения ученик увидит результат, а автор получит уведомление с деталями и статистикой.
